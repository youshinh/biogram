### **1\. ユーザーインターフェース設計: "AI Director Panel"**

画面中央またはFXパネル付近に配置する、AIへの指示出し専用パネル。

#### **A. メイン・トリガー (One-Action)**

* **\[ Deep Mix \-\> \] ボタン:** Deck A → B への定型ロングミックス（ユーザー設定依存）を即座に開始。  
* **\[ \<- Deep Mix \] ボタン:** Deck B → A へのミックス。

#### **C. 進行状況ビジュアライザー (Progress Monitor)**

AIが生成した「未来のカーブ」を可視化し、ユーザーに安心感を与える。

* **Waveform Overlay:** 波形の上に、これから動くフェーダーやフィルタの軌跡（オートメーションカーブ）を半透明の線で描画する。  
* **Phase Indicator:** 現在のフェーズを表示。  
  * \[ PRESENCE ... HANDOFF ... WASH OUT ... RESET \]  
  * 現在実行中のフェーズが点滅する。

### **2\. ユーザー体験フロー (UX Journey)**

1. **Awareness (認知):** ユーザーは次の曲をロードするが、どう繋ぐか迷っている、またはトイレに行きたい。  
2. **Direction (指示):** "AI Director" パネルを開き、「64 Bars」「Organic」「Tails」を選択してトリガーボタンを押す。  
3. **Generation (生成):**  
   * UI: "AI Architecting..." のローディング表示（1〜2秒）。  
   * System: GeminiからJSONを受信。バリデーションチェック。  
4. **Performance (実行):**  
   * 次の小節頭（Quantize）でミックス開始。  
   * UI上のノブやフェーダーが、まるで幽霊が操作しているかのように自動で動き出す（Ghost Fader）。  
   * ユーザーはそれを見ながら、気に入らない部分だけ\*\*「手動で介入（Override）」\*\*できる。  
     * ※ユーザーがノブを触った瞬間、そのパラメータだけAI制御から外れる設計が重要。  
5. **Completion (完了):**  
   * 曲が完全に入れ替わる。  
   * 前の曲の残響が美しく消えていく。  
   * 裏でパラメータがリセットされ、次の準備が整う。

このUXにより、ユーザーはAIを「勝手に動くロボット」ではなく、\*\*「信頼できるパートナー（Back-to-Backの相方）」\*\*として認識するようになる。
